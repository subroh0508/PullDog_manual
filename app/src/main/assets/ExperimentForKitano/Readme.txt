LinkList : 各タグの接続状態（距離含む）
    フォーマット : ID1, ID2, 距離
    備考 : 双方向に定義する必要あり。1と2が距離3でつながっているならば、1,2,3と2,1,3が必要。

LandmarkList : 屋内ランドマークの定義
    フォーマット : 対応タグID, 名前, リンカかどうか(1がリンカ)
    備考 : リンカには案内不可能

――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――

Landmark : 屋外ランドマークの定義
    フォーマット : 経度, 緯度, 名前
    備考 : 末尾に"_l"を付加すると、リンカになる。リンカには案内不可能。屋内ランドマークで定義したリンカと名前が同じならつながる。


――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――


TagList : 存在しているタグの定義
    フォーマット1 : ユニークID, タグID
    フォーマット2 : ユニークID, 緯度, 経度
    備考 : 現行ユニークIDとタグIDを同一にしてるけど、分ける必要ないかもしれない。事実フォーマット2はユニークIDのみで扱って動いている。とりあえず同じにしとけばバグは起きないと思われる。
           屋外タグを宣言すれば対応した緯度経度にMapがフォーカスするように作っているが、動作未確認。


 ――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――――


TagListDir : タグの方向テーブル
    フォーマット : 最初のタグID, 中間のタグID, 最後のタグID, この3つの順に進んだ際に曲がっている方向
    備考 : 方向の数字と方向の対応はRoute.javaにある通り。
            NAN       = -1;
            BOT       =  1;
            BOT_RIGHT =  2;
            RIGHT     =  3;
            TOP_RIGHT =  4;
            TOP       =  5;
            TOP_LEFT  =  6;
            LEFT      =  7;
            BOT_LEFT  =  8;

KitanoLine : 線字ブロックの定義
    フォーマット : 緯度, 経度, ....(任意個の緯度経度のセット), 名前
    備考 : 手書きで引いたやつ。実データに移行する際、T字の宣言に留意すること。（分割の必要あり）

KitanoPoint : 点字ブロックの定義
    フォーマット : 緯度, 経度, ....(任意個の緯度経度のセット), 名前
    備考 : 一応線字とファイルを分けているが、特に内部では分けてない。

Information : 付加情報
    フォーマット1 : ユニークID, 緯度, 経度, 内容, カテゴリー
    フォーマット2 : ユニークID, 緯度, 経度,　タグID, 内容, カテゴリー
    フォーマット3 : ユニークID, タグID, 内容, カテゴリー
    備考 : 屋内屋外各対応済み。カテゴリーの内訳はInformation.javaにある通り。
            TWEET        = 1;
            SYSTEM_INFO  = 2;
            SYSTEM_ALERT = 3;
            USER_INFO    = 4;
            USER_ALERT   = 5;

          Idを同じにすることによって、複数の類似情報を1度だけ提示できる。（横断歩道とか。渡り切った後はいらないので。）

SafeArea : 安全地帯。案内開始可能区域
    フォーマット(1行目) : "SafeArea"を記載すること。識別のため。
    フォーマット(2行目以降) : 緯度, 経度, ....(任意個の緯度経度のセット), 名前
    備考 : 必要か迷ったら定義しとくべき。
